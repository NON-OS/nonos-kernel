NONOS Agents Subsystem - Test Documentation
============================================
Date: 2026-04-04
Author: eKisNonos

OVERVIEW
--------
The agents subsystem provides autonomous task execution capabilities for the
NONOS kernel. This includes agent lifecycle management, task scheduling,
memory isolation, tool registration/execution, and configuration presets.

TEST MODULES
------------
8 test modules covering the agents subsystem:

1. core_tests.rs (248 lines)
   - Agent state machine variants (Idle, Running, Paused, Error, Complete)
   - Message role variants (System, User, Assistant, Tool)
   - AgentConfig defaults and custom configuration
   - Agent creation, message handling, clone operations
   - Name extraction with null terminators
   - Large message content handling (10KB+)

2. registry_tests.rs (211 lines)
   - Agent creation with unique ID generation
   - Agent retrieval by ID
   - Mutable agent access patterns
   - Agent listing and enumeration
   - Agent deletion and cleanup
   - Configuration preservation across operations
   - Agent isolation verification

3. context_tests.rs
   - Execution context management
   - Context isolation between agents
   - Context serialization/deserialization

4. memory_tests.rs
   - Agent memory allocation
   - Memory limits and bounds checking
   - Memory cleanup on agent termination

5. tasks_tests.rs
   - Task creation and queuing
   - Task priority handling
   - Task cancellation
   - Task status tracking

6. scheduler_tests.rs
   - Round-robin scheduling
   - Priority-based scheduling
   - Preemption handling
   - Idle agent detection

7. presets_tests.rs
   - Preset configuration loading
   - Custom preset creation
   - Preset validation

8. tools_tests.rs (172 lines)
   - Tool registration with name/description
   - Tool execution with arguments
   - Tool not found handling
   - Binary argument support
   - Large argument handling (10KB+)
   - Multiple tool registration
   - Tool listing with metadata

TEST EXECUTION
--------------
Tests require QEMU execution environment due to kernel target (x86_64-nonos).
The test framework uses #[test_case] attribute for kernel-mode test discovery.

To run tests:
  make test-agents

Or via QEMU directly:
  make qemu-test TESTS=agents
