The one open choice — determinism mechanism

Recommended: back-buffer + 1 atomic blit/commit

  • "No partial frame" true by construction — FB only ever shows fully-composed frames
  • ~8 MiB buffer allocated PRE-ExitBootServices (consistent with W4-08: no post-EBS alloc)
  • Failure renderer uses the same pre-reserved buffer (error-path alloc-free, cross-ref W5-05)

Rejected: damage-tracked region redraw

  • Tear-free only if every stage's damage set is exactly right — fragile
  • Tearing is precisely the failure mode we are eliminating
  • Saves memory we already proved is safe to spend
