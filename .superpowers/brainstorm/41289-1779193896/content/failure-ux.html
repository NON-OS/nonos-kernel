Failure screen — what does a legible, fail-closed halt look like?

All three are fail-closed: a verification failure (or menu timeout / invalid input) HALTS or selects the more-verified path — never a "boot anyway" affordance. They differ in how much passed-stage context they keep and how loud the halt is. Uses existing constants: ERROR #FF4444, ERROR_BG #1A0808, WARNING #FFAA00, teal #00D4AA for passed stages. Pick one.

⛔ BOOT HALTED
STAGE 6 · ED25519 VERIFY — FAILED
reason: signature does not match trusted key
key id: 0x00 · image: kernel.efi (4.1 MiB)
system is NOT booted · power-cycle to retry
no key → no boot · this screen has no "continue"

A · Full-screen halt card

Boot stops, screen is replaced by ONE centered card: failed stage, precise reason, posture ("NOT booted"), no continue. Maximum legibility, single deterministic frame, strongest fail-closed signal. Loses the passed-stage list.

✓ 1 UEFI   ✓ 2 SECURITY   ✓ 3 HARDWARE
✓ 4 KERNEL LOAD   ✓ 5 BLAKE3
✗ 6 ED25519 VERIFY — signature invalid (key 0x00)
· 7 ZK   · 8 ELF   · 9 HANDOFF  (not reached)
log ▸ …ed25519 verify start · FAIL · halt
▌ HALTED — fail-closed, not booted

B · In-place stage failure

Stage list stays; failed row turns red with inline reason; later stages shown un-reached; frozen log tail; halt banner. Preserves "what passed" forensic context. Busier; the halt is less of a hard visual stop.

✓1 ✓2 ✓3 ✓4 ✓5 ✗6 ·7 ·8 ·9
log ▸ ed25519 · FAIL
⛔ STAGE 6 ED25519 — HALTED
signature invalid · NOT booted
power-cycle to retry · no continue

C · Two-tier (context + halt overlay)

Failed stage marked red inline and a modal halt overlay on a dimmed stage list. Keeps forensic context AND a hard stop. Most informative; most drawing complexity (overlay compositing — must still be one committed frame, no partial composite).

My recommendation: A · Full-screen halt card. It is the cleanest single deterministic frame (no overlay compositing risk → trivially "no partial frame"), the loudest fail-closed signal, and the least attacker-controlled text on screen. The passed-stage context B/C preserve is better served by the serial log (which the audit already treats as the forensic channel) than by the framebuffer. If you want minimal forensic context on-screen, A can carry a one-line "passed: 1–5" subhead without becoming B.
